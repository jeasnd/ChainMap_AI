1. 为什么采用静态网站？

静态网站指服务器（或 CDN）只向访问者返回已经做好的 HTML、CSS、JS 和图片文件，不在服务器上为每个访问者临时执行复杂的数据库查询或登录逻辑。对你这种以文章与图谱介绍为主的站点，静态方案通常更简单、更便宜、加载更快，也更容易备份：整个文件夹打包带走即可。

你看到的“页面之间跳转”本质上是浏览器打开另一个 *.html 文件，或由本站的 JavaScript 做平滑滚动、菜单展开等——这些都不需要“动态网站”才能完成。动态网站一般指后台有数据库、用户系统、服务端渲染（如 PHP/Java/Node 每次现拼页面）等；当你需要用户注册、评论入库、电商下单时再考虑更合适。

2. 你要的“动态跳转”已经怎么做？

  • 多页面导航：index.html、article-memory.html、guide-hosting.html 之间用链接互相跳转，与百度、新浪等门户的“点菜单进不同栏目”是同一种机制。
  • 单页内锚点：首页里的「产业链」「投资视角」等，通过 #chain 这类锚点在长页面内平滑滚动。
  • 动效：styles.css 里的过渡与 script.js 里的菜单、滚动行为，让交互更顺，但文件仍是静态的。

若以后需要博客式后台，可再升级为 GitHub Pages + Hexo/Hugo 或 WordPress 托管，那是另一条路线，本仓库不必改架构也能先跑着。

3. 可实现的一步步路径：GitHub Pages + 自有域名

下面是一条从零到能用浏览器打开你的域名的常见路线。界面名称若略有出入，以 GitHub / 域名商当前页面为准。

阶段一：让网站先有一个公网地址（不绑域名也能测）

  1. 注册 GitHub 账号：打开 github.com/signup，按提示完成注册并登录。
  2. 新建仓库：右上角 “+” → “New repository”。仓库名可自取（例如 invest-site），选 Public → Create repository。
  3. 把本站文件放进仓库（二选一）：
    • 网页上传：仓库页点 “Add file” → “Upload files”，把电脑里整个网站文件夹里的 index.html、styles.css、script.js、各 article-*.html、guide-hosting.html 等拖到页面，Commit changes。
    • 用 Git 推送（熟悉命令行时）：本机安装 Git，在网站文件夹里执行 git init、在 GitHub 上复制仓库地址后 git remote add origin ...，再 git add .、git commit -m "init"、git push -u origin main（若默认分支是 master 则按仓库提示）。
  4. 开启 GitHub Pages：仓库 → Settings → 左侧 Pages → Build and deployment → Source 选 Deploy from a branch → Branch 选 main（或 master）且文件夹选 / (root) → Save。
  5. 等待构建并访问测试地址：几分钟后，同一 Pages 页面会显示站点地址，一般为 https://你的用户名.github.io/仓库名/。用浏览器打开，确认首页能显示。

若 404：确认仓库根目录（不是子文件夹里）有 index.html；GitHub 区分文件名大小写；首页地址末尾常需带仓库名路径，除非使用 用户名.github.io 专用仓库规则（可查 GitHub Pages 文档）。

阶段二：购买域名

  1. 在 阿里云、腾讯云、Cloudflare 等任一处搜索想用的名字（如 zhangsan-invest.com），下单并完成付款。
  2. 在域名控制台找到 DNS 解析 / 解析设置，准备添加记录。

阶段三：把 chainmapai.com 绑到 GitHub Pages

本站根目录已有 CNAME 文件，内容为 chainmapai.com。推送到 GitHub 后，在 Pages 里填同一域名即可。

  1. GitHub 仓库 → Settings → Pages → Custom domain 填入 chainmapai.com → Save。DNS 生效且证书就绪后，勾选 Enforce HTTPS。
  2. 在域名商 DNS 里新增（只绑 www 时）：类型 CNAME，主机记录 www，记录值填 你的GitHub用户名.github.io（与 GitHub Pages 说明一致）。
  3. 若还要支持根域名（example.com 不带 www）：在 GitHub Custom domain 中也可添加根域名；DNS 侧需按 GitHub 官方文档 配置 A 记录指向文档列出的 IP，或使用域名商提供的 ALIAS/ANAME 指到 用户名.github.io。IP 以文档为准，会更新。
  4. 等待生效：DNS 传播通常数分钟至数小时。成功后用 https://www.你的域名.com 应能打开与 GitHub Pages 相同的站点。

4. 路径 B：国内 OSS + 备案（概要）

若主要访客在中国大陆且使用国内 OSS 静态网站托管或国内节点 CDN，通常需要 ICP 备案。流程为：在阿里云/腾讯云等购买域名与大陆可用 OSS → 按控制台指引提交备案 → 通过后上传本仓库全部静态文件到 Bucket 并开启静态网站托管 → 在该厂商 DNS 把域名解析到 OSS 网站端点。细则以服务商当下页面为准。

5. 自己维护：从改一句话到发一篇新文章（逐步）

以下以 GitHub 为例；若你用手动上传到 OSS，把 “push 后自动更新” 理解为 “上传覆盖文件”。

5.1 只改首页或某篇文章里的文字

  1. 用 Cursor 或 VS Code 打开你的网站文件夹（与 index.html 同级）。
  2. 打开要改的 HTML，例如 index.html 或 article-memory.html。
  3. 只改标签之间的正文，避免删掉 <div>、</p> 等成对标签。
  4. 保存（Ctrl+S）。
  5. 用 Git 时：在该文件夹打开终端，执行 git add . → git commit -m "更新文案" → git push。约 1～3 分钟后刷新线上地址。
  6. 用网页编辑时：在 GitHub 打开对应文件 → 铅笔 Edit → 保存 Commit。

5.2 新增一篇专题文章（推荐按此顺序做）

  1. 复制 article-memory.html，重命名为新英文名，例如 article-chip.html（勿用空格与中文文件名）。
  2. 打开新文件，修改：<title>、meta description、<h1>、article-body 内正文；目录 article-toc 里每个 href="#xxx" 与正文里 <h2 id="xxx"> 一一对应。
  3. 打开 index.html，在 <div class="article-cards"> 里复制一整块 <a class="article-card" ...>，改 href="article-chip.html" 与卡片标题、摘要。
  4. （可选）在其它页面的 <nav class="nav"> 里加一条指向新文章的链接。
  5. 本地双击 index.html 用浏览器点链接自测。
  6. git add / commit / git push（或上传全部变更文件）。

5.3 两个原则

  • 相对路径：链接写 href="article-chip.html"、href="styles.css"，不写本机盘符。
  • 备份：大改前复制整个网站文件夹一份。

本站页面均使用相对路径；保持文件夹结构不变，整包上传到托管根目录即可。
