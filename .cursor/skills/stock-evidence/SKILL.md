# 个股论证 · 页面与数据结构

## 页面目标

用户点击股票后应能回答：**为什么是这个方向里优先研究它？依据是什么？风险在哪？**

## 必备内容块（对应 `stock.html` 渲染）

1. **结论摘要** — `oneLiner` + `rank` + 市场/代码
2. **产业链位置** — 在六层中的哪一环、与热点 `topicId` 的关系
3. **选股理由** — `thesis` 数组（3～5 条，每条可核对）
4. **壁垒拆解** — `moatTags` + 可选 `moatDetail`
5. **关键数据** — `metrics` 表格（指标 / 数值 / as-of / source）
6. **证据链接** — `evidence[]`：`{ "type": "财报|行业|新闻", "title", "url", "date" }`
7. **风险与反面** — `risks[]`
8. **合规** — 固定免责声明 + 数据日期

## thesis 写作模板

每条一句，结构：**事实 → 含义 → 与 AI 投资逻辑的关系**

示例：

> 公司是当前 HBM 主要供应商之一，供给紧张时议价能力提升，与 AI 算力 capex 周期同向。

## metrics 最低集合（能填则填）

| 指标 | 用途 |
|------|------|
| 营收同比增速 | 景气 |
| 毛利率 / 营业利润率 | 盈利能力 |
| ROE | 资本效率 |
| 研发费用率 | 技术投入（科技股） |
| 市占或产能描述 | 壁垒（定性也可） |

缺失时写 `—` 并注明「待从下一季财报更新」。

## evidence 质量

- 优先：上市公司公告、财报 PDF、交易所披露
- 其次：一手行业机构摘要（标注机构名与日期）
- 避免：无出处社交帖子作为唯一依据

## JSON 扩展示例

```json
"thesis": ["...", "..."],
"moatDetail": "一段话展开壁垒",
"evidence": [
  { "type": "财报", "title": "FY2025 Q2 业绩说明", "url": "https://...", "date": "2025-03" }
],
"risks": ["存储价格周期性回落", "资本开支不及预期"]
```

## Agent 维护

每次改 `thesis` 或 `metrics` 时同步更新 `stocks.json` 顶层 `updatedAt` 及对应 topic 的 `updatedAt`。
