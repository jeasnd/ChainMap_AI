# 选股规则 · 每方向 1～3 只

## 数量与市场

| 规则 | 说明 |
|------|------|
| 数量 | 每个 `topicId` **1～3 只**，宁缺毋滥 |
| 市场 | `market`: `US` \| `A` \| `HK`，同一方向尽量覆盖不同市场各 0～1 只 |
| 排名 | `rank`: 1 为框架内首选，2、3 为备选或区域龙头 |

## 「最值得投资」在本站的定义

**不是**预测短期涨幅，而是**研究优先级最高**：

1. **壁垒最高**：技术/份额/生态护城河可陈述
2. **最赚钱**：毛利率、净利率、ROE 相对同业领先（有数据则写进 `metrics`）
3. **最难替代**：客户切换成本高、标准绑定、产能稀缺

若 A/HK 无纯正标的，可：

- 选产业链映射最清晰的（如光模块、IDC、电力设备）
- 或该市场不列入，**不凑数**

## 跨市场映射示例（存储层）

| 市场 | 示例逻辑 | 常见候选类型 |
|------|----------|----------------|
| US | 直接 HBM/DRAM 敞口 | MU |
| 非美上市 | HBM 龙头 | SK hynix（韩股，站点可标 `KR` 或链到说明） |
| A | 模组/封测/设备，非纯正 HBM | 谨慎标注「间接敞口」 |
| HK | 常与美股/韩股重复 | 若无则用 0 只 |

## JSON 核心字段（`data/stocks.json`）

```json
{
  "id": "mu",
  "topicIds": ["storage-hbm"],
  "rank": 1,
  "market": "US",
  "symbol": "MU",
  "name": "美光科技",
  "nameEn": "Micron Technology",
  "oneLiner": "一句话选股结论",
  "moatTags": ["HBM产能", "与英伟达绑定"],
  "metrics": [{ "label": "毛利率", "value": "—", "asOf": "2026-Q2", "source": "财报" }],
  "stockIds": []
}
```

## 禁止

- 无主营业务关联的纯概念炒作股作为 rank 1
- 同时推荐 >3 只导致焦点分散
- 未核实财报数字仍写精确百分比

## 与宏观层联动

更新顺序建议：**先**调 `hot-topics.json` 的 `stockIds`，**再**确保 `stocks.json` 里对应条目 `topicIds` 双向一致。
